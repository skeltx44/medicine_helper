# 🌐 올바른 접속 방법

## ❌ 잘못된 방법 (하지 마세요!)

**main.html 파일을 직접 더블클릭하면:**
- 파일 경로가 `file:///C:/Users/ADMIN/Desktop/sibal/main.html` 형태로 됩니다
- 이렇게 하면 API 호출이 작동하지 않습니다!
- CORS 오류가 발생할 수 있습니다

## ✅ 올바른 방법

### 1. 백엔드 서버가 실행 중인지 확인
터미널에서:
```
 * Running on http://0.0.0.0:5001
```
이런 메시지가 보여야 합니다.

### 2. 프론트엔드 서버가 실행 중인지 확인
다른 터미널에서:
```
Serving HTTP on 0.0.0.0 port 8080
```
이런 메시지가 보여야 합니다.

### 3. 브라우저에서 접속

**Chrome, Edge, Firefox 등 브라우저를 열고:**

주소창에 입력:
```
http://localhost:8080
```

또는

```
http://localhost:8080/index.html
```

## 🎯 접속 순서

1. **로그인 화면** (`http://localhost:8080/index.html`)
   - 성명과 전화번호 입력 (아무거나 입력 가능)
   - 로그인 버튼 클릭

2. **메인 화면** 자동 이동
   - 검색, 복약 내역, 오늘의 약, 이달의 약 버튼이 보입니다
   - 챗봇 버튼도 보입니다

## 🔍 확인 방법

### 백엔드가 정상 작동하는지 확인:
브라우저에서:
```
http://localhost:5001/api/health
```

**정상이면:**
```json
{
  "status": "healthy",
  "timestamp": "..."
}
```

### 프론트엔드가 정상 작동하는지 확인:
브라우저에서:
```
http://localhost:8080
```

**정상이면:**
- 메인 화면 또는 로그인 화면이 보입니다

## ⚠️ 문제 해결

### "localhost에 연결할 수 없습니다" 오류
- 프론트엔드 서버가 실행 중인지 확인
- 터미널에서 `python -m http.server 8080` 실행 확인

### "서버에 연결할 수 없습니다" 오류 (앱 내에서)
- 백엔드 서버가 실행 중인지 확인
- 터미널에서 `python app.py` 실행 확인
- `http://localhost:5001/api/health` 접속 테스트

### CORS 오류
- 백엔드 서버가 실행 중인지 확인
- `backend/app.py`에서 CORS가 활성화되어 있는지 확인

## 📝 요약

**절대 하지 말 것:**
- ❌ main.html 파일을 직접 더블클릭
- ❌ 파일 탐색기에서 HTML 파일 열기

**반드시 해야 할 것:**
- ✅ 터미널에서 `python -m http.server 8080` 실행
- ✅ 브라우저에서 `http://localhost:8080` 접속
- ✅ 백엔드 서버도 실행 중이어야 함 (`python app.py`)
