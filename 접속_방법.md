# 🌐 올바른 접속 방법

## ❌ 잘못된 방법 (하지 마세요!)

**main.html 파일을 직접 더블클릭하면:**
- 파일 경로가 `file:///C:/Users/ADMIN/Desktop/sibal/main.html` 형태로 됩니다
- 이렇게 하면 API 호출이 작동하지 않습니다!
- CORS 오류가 발생할 수 있습니다

## ✅ 올바른 방법

> **⚠️ 중요:** 백엔드는 Render에 배포되어 있습니다 (`https://sibaljom.onrender.com`)
> 프론트엔드는 로컬에서 실행하지만, API는 Render 서버를 호출합니다.

### 1. 프론트엔드 서버가 실행 중인지 확인
다른 터미널에서:
```
Serving HTTP on 0.0.0.0 port 8080
```
이런 메시지가 보여야 합니다.

### 3. 브라우저에서 접속

**Chrome, Edge, Firefox 등 브라우저를 열고:**

주소창에 입력:
```
http://localhost:8080
```

또는

```
http://localhost:8080/index.html
```

## 🎯 접속 순서

1. **로그인 화면** (`http://localhost:8080/index.html`)
   - 성명과 전화번호 입력 (아무거나 입력 가능)
   - 로그인 버튼 클릭

2. **메인 화면** 자동 이동
   - 검색, 복약 내역, 오늘의 약, 이달의 약 버튼이 보입니다
   - 챗봇 버튼도 보입니다

## 🔍 확인 방법

### 백엔드가 정상 작동하는지 확인 (Render):
브라우저에서:
```
https://sibaljom.onrender.com/api/health
```

**정상이면:**
```json
{
  "status": "healthy",
  "timestamp": "..."
}
```

> **참고:** 백엔드는 Render에 배포되어 있으므로 로컬에서 실행할 필요가 없습니다.
> 프론트엔드는 자동으로 `https://sibaljom.onrender.com`의 API를 호출합니다.

### 프론트엔드가 정상 작동하는지 확인:
브라우저에서:
```
http://localhost:8080
```

**정상이면:**
- 메인 화면 또는 로그인 화면이 보입니다

## ⚠️ 문제 해결

### "localhost에 연결할 수 없습니다" 오류
- 프론트엔드 서버가 실행 중인지 확인
- 터미널에서 `python -m http.server 8080` 실행 확인

### "서버에 연결할 수 없습니다" 오류 (앱 내에서)
- Render 서버가 정상 작동하는지 확인: `https://sibaljom.onrender.com/api/health`
- Render 서버는 첫 요청 시 약간의 지연이 있을 수 있습니다 (Cold Start)
- 네트워크 연결 상태 확인

### CORS 오류
- Render 서버가 정상 작동하는지 확인: `https://sibaljom.onrender.com/api/health`
- Render 서버의 CORS 설정 확인
